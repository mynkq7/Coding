# Contributing Guidelines

This repository follows simple, clean rules to keep learning material organized.

---

## Commit Message Style

Use clear prefixes:

- `feat:` new feature or new day
- `fix:` bug fix or correction
- `docs:` documentation changes
- `chore:` cleanup or refactoring

Example:
feat: add Day-04 looping exercises

---

## Folder Naming

- Use `Day-01`, `Day-02`, etc.
- Keep folder names consistent
- Use lowercase for file names unless required

---

## Code Style

- Add comments where helpful
- Keep files readable and simple
- Follow language best practices where possible

---

This repository prioritizes **clarity over cleverness**.
